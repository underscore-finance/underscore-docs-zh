# 收款人 (Payee)：赚取收益的自动支付

还在手动转账？每个月重复输入地址，担心输错？

收款人（Payee）是经过预先批准的收款地址，可从你的 Underscore 钱包接收支付——支付金额完全在你的掌控之内。直接发送支付或让收款人自行拉取资金，灵活自如。为每个人设置个性化限额，例如：员工 A 每月 $5,000（约3.5万人民币），员工 B 每月 $10,000（约7万人民币）。配置周期重置：每日、每周、每月或自定义。最棒的是，你的资金在 DeFi 中持续赚取收益，直至需要支付的那一刻。

用可编程支付关系替代传统电汇。自动发放薪资，同时资金持续复利增长。让承包商自行提取已批准金额，无需手动转账。轻松处理房租、订阅和各种定期账单。每笔支付都经过智能合约验证，每分钱在发送前都在为你赚取收益。

## 为什么需要收款人

### 加密支付的现实困境

目前进行定期加密货币支付，你只能在这些糟糕的选项中选择：

- **手动转账**：每次都要重新输入地址，担心手误，浪费宝贵时间
- **授予完全访问权**：分享私钥或授予无限支出权限，风险极高
- **传统银行**：支付高昂电汇费，失去 DeFi 收益，还要等待处理

这些方案都无法满足现代企业和个人对可靠、可控自动化支付的需求。

### 收款人解决方案

收款人功能创建一个经过验证的收款人列表，配置精细控制：

```
传统方式：给供应商信用卡 → 他们想收多少收多少，想什么时候收就什么时候收
Underscore：添加供应商为 Payee → 每月 $5,000 限额 → 仅限 USDC → 每季度自动过期
```

你的资金始终在 DeFi 协议（如 Aave、Morpho、Euler）中赚取收益，直至支付时刻。系统仅在需要时自动提取所需金额。此外，你的所有 DeFi 活动都能获得[奖励](rewards.md)。

## 为何你的信任圈坚不可摧

### 仅你掌握密钥

**绝对的所有者控制**：作为[钱包所有者](user-wallet.md)，仅你本人可向信任圈添加或移除地址。不是你的会计师，不是你的管理员，更不是你的 AI 助手。这一核心规则由区块链本身强制执行——这不是可被绕过的政策，而是无法被破解的代码。

### 设计上防篡改

**不可篡改的记录**：一旦你将某人添加到信任圈，该记录将永久写入区块链。没有黑客能秘密将「Alice 的钱包」改为指向自己的地址。你保存的地址就是永久不变的地址。

### 管理员（Manager）只能向你的信任圈支付

**最终安全保障**：当你授予管理员支付权限时，他们只能向已存在于你信任圈内的地址转账。他们无法添加新收款人，无法修改现有收款人，更无法意外地将你的整个金库转至未知地址。每笔支付都必须发送给你已经验证并批准的地址。

这不仅仅是安全措施——这是安全架构。你的信任圈在你的资金周围建立了一道坚不可摧的防线。

## 收款人（Payee）控制与配置

为每个 Payee 设置全面的控制措施，确保支付严格按照预期进行。这些控制措施防范超支、时间错误及未经授权的更改。

### 金融限额

#### 双重保护系统

每个 Payee 都支持代币金额和美元价值双重限制：

- **交易限额**：单笔支付的最大金额（例如 1,000 USDC 或 $1,000）
- **周期限额**：指定时间段内的最大金额（例如 5,000 USDC 或每月 $5,000）
- **累计限额**：总累计最大金额（例如 50,000 USDC 或 $50,000）

最严格的限额始终生效，防范价格波动风险。

#### 周期配置

- **自定义周期**：设置任意时长（以区块为单位，例如 Base 网络上 43,200 区块 ≈ 1 天）
- **自动重置**：限额在每个周期刷新（未使用的金额不会滚动）
- **常用设置**：每日、每周、双周、每月、每季度

### 支付控制

#### 交易限制

- **最大交易次数**：限制每个周期内的支付次数（例如月薪仅 1 次）
- **冷却期**：两次支付之间的最短间隔（防止意外重复支付）
- **到期日期**：在设定时间后自动停用 Payee（例如承包商合同结束）

#### 资产限制

精确控制每个 Payee 可接收的代币类型：

- **单一资产**：仅限一种代币（例如工资仅用 USDC）
- **首选资产**：设置默认值，灵活接受其他资产
- **任意资产**：接受所有代币（适用于交易台（trading desk））

### 安全功能

#### 全局设置（钱包范围）

- **拉取支付主开关**：一键启用/禁用所有拉取支付
- **自我支付开关**：允许/阻止向自己支付
- **默认限额**：新 Payee 继承的基础设置

#### 个性化覆盖

- **自定义限额**：为特定关系覆盖全局设置
- **拉取权限**：即使全局禁用，也可按 Payee 单独启用
- **资产白名单**：限制超出全局设置的特定代币

### 系统限制

- **最多 40 个 Payee**：每个钱包的容量上限
- **最多 40 个资产**：每个 Payee 的限制列表
- **故障保护**：价格预言机不可用时自动阻止支付
  - **保持启用**：防止价格源故障导致钱包被掏空
  - **仅在纯稳定币操作时禁用**：当你完全信任 1:1 锚定时

价格预言机为区块链提供真实世界资产价格。当价格数据不可用时阻止支付，系统可防止预言机故障期间的潜在损失。

### 拉取支付安全

**硬性要求**：你*无法*在未设置限额的情况下启用拉取支付。系统强制要求至少一项上限（交易或周期限额），杜绝无限制访问。

## 权限系统

### 谁可以添加 Payee

**所有者**：完全控制权，可添加、修改或移除任何 Payee  
**管理员**：可建议新 Payee（需所有者在延迟后批准）  
**其他人**：无权修改信任圈

### 谁可以执行支付

**所有者**：可在限额内向任何活跃 Payee 支付  
**管理员**：与所有者相同的支付权限（仅限活跃 Payee）  
**Payee**：若明确启用，可主动拉取支付

### 谁可以移除

**所有者**：可立即移除任何 Payee  
**Payee**：可自行移除（干净退出）  
**管理员**：无法直接移除 Payee

## 生命周期管理

### 激活流程

```
已添加 → 待处理 → 活跃 → 过期/已移除
        （延迟）  （工作中）
```

| 用例         | 激活延迟 | 自动过期 | 为何选择此组合                 |

| **正式员工** | 3 天     | 365 天   | 中等安全级别；配合年度合同     |
| **家人亲属** | 1 天     | 永不     | 紧急情况快速访问；永久信任关系 |
| **订阅服务** | 2 小时   | 30 天    | 快速设置；配合月度账单周期     |

### 管理层级

**所有者（你）**

- 直接添加/更新/移除
- 确认管理员提案
- 始终拥有完全控制权

**[管理员](managers.md)** — 你的可信操作员

- 可向你信任圈内的任何人支付
- 可建议新 Payee（需你批准）
- 无法直接修改你的信任圈
- 完美平衡控制权与执行效率

**Payee**

- 可自行移除
- 实现干净退出

> **📋 管理员提案需你审批**  
> 管理员提出新增 Payee 后，你将看到待处理请求。经过安全延迟（可自定义）后，你必须手动确认才能激活。这种两阶段验证可防止未经授权的添加。

## 拉取支付：订阅支付革新

允许 Payee 直接从你的钱包拉取支付，始终在你预设的限额内运行。

### 工作原理

1. Payee 发起提取请求
2. 智能合约验证所有配置限额
3. 若需要，自动从收益头寸调用资金
4. 限额内立即执行支付

### 需要双重激活

```
全局: canPull = true  且  Payee: canPull = true
                ↓
         拉取支付已启用
```

### 实际收益对比

传统订阅模式：

- 每月 $500 闲置 = $0 收益

使用拉取支付：

- 每月 $500 赚取 5% 年化收益率（APY） = 每年额外 $25（约 175 元人民币）
- 年度订阅 $6,000 = 每年奖励 $300（约 2,100 元人民币）

**安全保障**：拉取 Payee 必须设置限额，杜绝无限制访问。

## 常见问题

**到期后会发生什么？**  
Payee 自动停用。支付被阻止直至你手动续期。

**如果误付了多余金额怎么办？**  
不可能发生。智能合约强制执行硬性限制，超过配置金额的交易会被自动拒绝。

**需要为每位员工设置独立的 Payee 吗？**  
是的。每个 Payee 有独立的限额和跟踪记录，确保精确控制。

**设置后能否修改限额？**  
可以。更改立即生效，无需等待。

**Gas 费如何处理？**  
系统自动计算 Gas 费，从你的钱包余额中扣除，不影响支付金额。

## 完美组合：管理员 + Payee

想象一下这样的场景：

**你的信任圈**（Payee）定义了谁可以接收支付。  
**你的操作员**（[管理员](managers.md)）负责执行日常支付。

共同构建坚不可摧的安全体系：

- 你只需验证地址一次（添加为 Payee）
- 管理员处理日常运营（仅向这些 Payee 支付）
- 你保留最终控制权（仅你可修改信任圈）

这种权责分离意味着你可以委托工作而不委托信任。你的 CFO 可以支付所有供应商，却无法偷偷添加他们的亲戚为新「供应商」。你的 AI 助手可以优化支付流程，却无法将资金转到未知地址。
