# 数字支票 (Cheque)：立即发送，稍后确认

数字支票是带时间锁的付款承诺，让你在兑现前拥有完全控制权。向承包商开具 $5,000 支票——如果对方未履约，随时取消。为大额支付设置 72 小时安全延迟。收款人立即看到支票，但需等待延迟期结束才能兑现。你的资金在被领取前持续产生收益。

适用于承包商付款、需要审核时间的大额转账，或任何需要撤销功能的支付场景。无需预先批准——可发送至任何地址。兑现前随时取消。双方均可在链上查看完整透明记录。

## 为什么需要数字支票

### 即时加密支付的问题

传统加密转账在点击发送后立即不可逆转。没有宽限期。无法撤回。这为合法交易带来不必要的风险，例如：

- 需要在付款前验证工作成果
- 大额转账需要额外安全措施
- 支付时间需与业务流程同步
- 地址或金额错误必须可更正

### 数字支票解决方案

数字支票将支付授权与执行分离：

- **先写后付**：创建按你的条件执行的支付承诺
- **兑现前可取消**：在资金转移前始终保持控制权
- **自动安全延迟**：大额支付内置审核期
- **收益保留**：资金在支付前持续在钱包中产生收益，并可获得[协议奖励](rewards.md)
- **一次性简便**：无需管理重复计划——适合单笔支付

## 四阶段生命周期

```
创建 → 时间锁 → 活跃 → 完成
  │      │       │      │
  └──────┴───────┴──────┴─→ 支付前随时可取消
```

### 开具（创建）

就像填写纸质支票，你指定收款人和金额。我们记录此刻的美元价值——相当于在支票上写今天的日期。资金尚未转移；你只是做出了支付承诺。

### 日期设定（时间锁）

还记得远期支票吗？我们的安全延迟原理相同：

- 大额付款会自动采用「远期支票」方式以提供保护
- 你可为未来支付设置自定义日期
- 资金留在账户中赚取利息，就像在银行一样

### 可兑现（活跃）

支票现在有效，可通过以下方式兑现：

- 你决定立即处理
- 你的[管理员](managers.md)处理
- 收款人自行领取（如果你启用了此功能）

### 已清算或作废（完成）

就像实体支票，最终结果为：

- **已清算**：资金成功转至收款人
- **作废**：过期未兑现或被你取消

## 智能安全功能

### 自动美元阈值

设置单一延迟阈值（如 $1,000）：

- 低于：支票可立即处理
- 高于：自动应用延迟

> **注意**：无法为**[白名单](whitelist.md)**地址创建支票。如果你足够信任某地址而将其加入白名单，请使用白名单功能进行即时无限转账——支票用于需要支付控制的地址。

示例影响：

- $500 支付 → 无需延迟
- $5,000 支付 → 应用安全延迟

### 时间控制

- **高额延迟**：高价值支票等待时间（如 3-7 天）
- **默认有效期**：支票保持活跃时长（如 30 天）
- **自定义时间**：可按需为每张支票自定义时间设置

时间以区块计算（在 Base L2 上：约 2 秒/区块，约 43,200 区块/天）。

## 财务控制

你可以设置多种控制和限制，确保数字支票完全按你的需求运作。这些控制帮助你管理风险、防止错误并监督支付操作。无论你是管理个人财务的个人用户，还是处理供应商付款的企业，都可以根据具体需求和风险偏好自定义这些设置。

### 创建限制

- **最大活跃支票数**：允许的未支付支票总数（如 20 张）
- **最大支票金额**：单张支票最大值（如 $50,000，约35万人民币）
- **周期上限**：月度创建限额（如 $100,000）

### 支付限制

- **周期支付上限**：每周期可兑现的最大金额（如 $50,000/周，约35万人民币）
- **支付冷却期**：支付间隔时间（如 10 分钟）

### 资产限制

将支票限制为特定代币：

- 仅使用稳定币（保守操作）
- 任何代币（满足更灵活的业务需求）
- 单一资产简化会计

## 权限系统

### 谁可以创建

**所有者**：始终可创建任何支票
**管理员**：仅在全局启用且已为其单独授予许可时

### 支付选项

**推送支付**：由你或管理员执行
**拉取支付**：收款人准备好时领取（必须明确启用）

### 谁可以取消

**所有者**：支付前任何支票随时可取消
**安全管理员**：仅具有紧急取消权限

## 实际配置示例

### 社交支付

```
金额：$30 USDC

有效期：7 天
拉取：启用
结果：朋友方便时领取午餐费
```

### 自由职业者发票

```
金额：$3,000 USDC
解锁：3 天（超过 $1,000 阈值自动）
有效期：30 天
拉取：启用
结果：客户展示承诺，自由职业者控制时机
```

### 月度房租

```
金额：$2,500 USDC
创建：每月 25 日
解锁：次月 1 日
拉取：启用
结果：房东按时收款，你保持控制
```

### 高价值合同

```

解锁：7 天（高额延迟）
有效期：45 天
审核优势：验证交付成果的时间
延迟期间收益：约 $48*
```

\*收益随当前市场利率变化

## 收款人体验

数字支票的设计让收款人享受与发送方同样简便安全的体验。收款人享有完全透明度——在提供商品或服务前可在区块链上验证支票的存在和条款。安全延迟（如有）结束后，收款人可在有效期内随时领取付款。

这营造了一种建立信任的互动：收款人看到你的支付承诺，而你保留出现问题时取消的能力。区块链确保双方随时清楚了解支付状态，消除了传统支付方式常见的不确定性。

## 常见问题

**如果我发送了错误金额怎么办？**  
在支票兑现前取消，创建新的。

**收款人能看到待处理的支票吗？**  
是的，从创建起在链上完全透明。

**我可以修改支票吗？**  
不可以，但你可以取消并用新条款重新创建。

**我可以向白名单地址开支票吗？**  
不可以——使用[白名单](whitelist.md)进行即时支付。支票用于需要支付控制的非白名单收款人。

**支票过期后会怎样？**  
就像银行的作废支票，过期支票自动失效。资金从未离开你的钱包，因此立即可用于其他用途。无需你采取任何行动。

**支票金额或数量有限制吗？**  
有的，出于安全考虑。你可以设置每张支票的最大值和活跃支票总数限制。这些保障措施防止意外过度承诺，同时为正常使用提供充足灵活性。

**我的[管理员](managers.md)可以处理支票吗？**  
可以，而且非常实用。你可以授权[钱包管理员](managers.md)在你设定的限额内创建和取消支票。这对于需要 CFO 处理供应商付款的企业，或希望财务顾问协助管理大额交易的个人来说非常理想。优势在于管理员只能在你预设的边界内操作——他们不能超出支出限额或更改你的安全设置。了解如何在我们的指南[使用管理员管理你的钱包](managers.md)中设置这些安全委托权限。
