# Earn Vaults：面向所有人的 AI 管理收益

Underscore Earn Vaults 是将 AI 管理的收益策略封装成标准 ERC-4626 代币的产品。你存入资产，获得代表自己在主动管理组合中份额的 Vault 份额代币。Vault 的 AI 代理会全天候监控收益率，捕捉利率高点、监测流动性风险、领取奖励，并在多个协议之间再平衡，而你只需要持有这个代币。

Vaults 是非托管的。资产始终属于你。你可以随时提取，无需任何额外许可。

它与[可编程钱包](user-wallet.md)最大的不同在于：你的头寸不会被锁死。Vault 份额是可组合代币，可以在整个 DeFi 中使用，你可以交易、将其作为抵押品，或跨链桥接。无论这些代币被转移到哪里，底层 AI 策略都会继续运行。

## 为什么会有 Vaults

### 起源

Vaults 并不是凭空设计出来的，而是源于我们在实际构建过程中遇到的真实问题。

我们最早从 [Hightop](https://hightop.com) 开始。在那里，AI 代理通过独立的[可编程钱包](user-wallet.md)管理用户投资组合。AI 会分析整个 DeFi 世界中的收益机会，然后对每个用户的钱包做再平衡，以抓住更优机会。它确实能工作，但我们很快撞上了天花板：每一次优化都意味着要触碰每个钱包。如果我们想把 10,000 个用户迁移到 Morpho 中一个更高收益的市场里，就意味着 10,000 笔独立交易。成本高、速度慢，而且 AI 对那些目标一致的用户重复做着同一份工作。

随后是 Ripe Protocol。用户会拿自己的生息头寸作抵押品，例如 Aave 存款、Morpho 头寸等等。但一旦这些资产变成抵押品，它们就被锁住了。即便后来出现更优收益机会，用户也无法在不先还清贷款的情况下重新平衡这些抵押品。那些本应继续生息的资产就这样卡住了。

这两个问题都来自同一个地方：真实用户、真实使用场景、真实反馈。我们在 Hightop 上亲身体会到了，把 AI 优化扩展到大规模用户时的低效。

于是我们做了 Vaults。核心洞察是：保留我们在可编程钱包里已经建立起来的一切，AI 能力、清晰边界、链上规则和策略，但只给 AI 管理一个“钱包”。用户把资金存入一个共享 Vault，收到代表份额的代币。AI 优化一次，所有人都受益。由于 Vault 份额是标准代币，它们还能在 Ripe 上作为抵押品使用，而底层 AI 会继续优化这些头寸。

这种架构也打开了通往更多 Vault 类型的大门：杠杆策略、指数型 Vault 等，都可以建立在同一套基础之上。

如今，Vaults 已经上线，并在 Hightop 中投入生产使用。

### 可扩展性

一个 Vault，一次再平衡，所有人受益。

这就是我们扩展到数百万用户的方式。

### 可组合性

Vault 份额是标准 ERC-20 代币。这带来了：

- **抵押能力**：在 Ripe Protocol 上使用 Vault 份额借款，而 AI 会继续优化你的抵押品，即便它已经被锁定
- **流动性**：可在流动性池中出售份额，实现即时退出
- **可迁移性**：可跨链桥接，也可在其他协议中使用
- **易集成**：任何应用都可以通过标准接口集成 Vaults

你的头寸会变成积木，而不是被锁住的资产。

### 简洁性

追逐收益是一件很累的事。盯着 Morpho 的利率、比较 Aave 市场、追踪 Euler 的激励活动、赶在奖励过期前领取收益，这几乎是一份全职工作。

Vaults 把这些全部接管了。用户只需要存入资产、拿到一个代币，然后赚取收益。所有复杂度，包括多协议策略、AI 再平衡、风险管理、奖励领取和复投，都会在幕后完成。

## Earn Vaults 如何运作

为你的资产提供简单的收益优化。

### 工作方式

1. **存入**：把 USDC、ETH 或其他支持资产存入 Vault
2. **获得份额**：收到代表你在池中份额的 Vault 代币（例如 `_USDC`）
3. **AI 优化**：代理在已批准的协议、Vault 和收益策略之间进行再平衡
4. **随时提取**：任何时候都可以把份额赎回成底层资产

### AI 在做什么

AI 代理会持续优化 Vault 的头寸：

- **持续监控**：跟踪 Morpho、Euler、Moonwell、Aave、Fluid、Compound 等协议的利率、流动性、激励和提取条件
- **风险调整后的路由**：不只看名义 APY，还会分析存款人数、总存款、利用率和可提取流动性，再决定是否迁移资金
- **自动再平衡**：随着条件变化，把资金迁移到风险收益比更优的地方
- **奖励收割**：自动领取并复投协议激励

### 示例：USDC Earn Vault

**你存入 10,000 USDC**：
- Vault 总规模为 1,000,000 USDC，份额价格为 1.00 美元
- 你收到 10,000 枚 `_USDC`

**3 个月后**：
- AI 已执行数十次再平衡，捕捉利率机会、在迁移前评估风险、轮动参与激励活动，并在奖励解锁后及时领取
- Vault 现在总共持有 1,025,000 USDC（增长 2.5%）
- 你的 10,000 枚 `_USDC` 现在可赎回 10,250 USDC
- 你无需进行任何操作

## 与 Ripe Protocol 结合的资本效率

Vault 份额可以在 [Ripe Protocol](https://ripe.finance) 中成为高质量抵押品，这会带来更强的资本效率：

1. **存入** Earn Vault，获得 `_USDC`
2. **把 `_USDC` 作为抵押品** 存入 Ripe 借出稳定币
3. **你的抵押品继续生息**，AI 会在后台持续优化收益
4. **把借出的稳定币继续部署** 到其他用途

你的抵押品会 24/7 持续工作。它不是躺着不动的静态抵押物，而是在你使用杠杆的同时继续赚钱。

## 杠杆 Vaults

通过在 Ripe Protocol 上使用受控杠杆来获得更高收益，同时内置债务安全机制。

### 工作方式

杠杆 Vault 通过抵押借款来放大收益，但它有一个关键安全特性：**借入资金始终保持美元计价敞口**，以确保 Vault 总能偿还债务。借来的金额也会再次作为额外抵押品存回 Ripe，从而形成两层抵押支持：你的原始资产 + 借来的美元资产。

完整流程如下：

1. **你存入** 一个波动型资产到 Underscore 杠杆 Vault（例如 cbBTC、ETH）
2. **Vault 把你的资产** 存入对应的 Earn Vault（由 AI 优化收益）
3. **Vault 将这些生息份额** 作为抵押品存入 Ripe Protocol
4. **Vault 以该抵押品借出** GREEN（稳定币）
5. **Vault 将 GREEN 兑换为** USDC
6. **Vault 把 USDC** 存入 USDC Earn Vault（继续赚收益）
7. **Vault 再把这些 USDC Vault 份额** 作为额外抵押品加入 Ripe

结果是：你的原始资产在生息，而借来的那部分资金也在生息；同时，借来的这部分始终保持在美元计价资产中。

#### 收益测算：杠杆 cbBTC

假设你存入价值 100,000 美元的 cbBTC：

| 层级 | 金额 | 利率 | 年化收益 |
|-------|--------|------|--------------|
| cbBTC 在 Earn Vault 中 | 100,000 美元 | 0.5% | +500 美元 |
| 从 Ripe 借款（70% LTV） | 70,000 美元 | 3%* | -2,100 美元 |
| USDC 在 Earn Vault 中 | 70,000 美元 | 8% | +5,600 美元 |
| **净收益** | | | **+4,000 美元** |

*\*Underscore Vaults 可获得折扣借款利率（3%，而标准利率为 6%）*

**综合 APY：4.0%**。对于你原本的 100,000 美元 cbBTC 来说，相比基础 0.5% 收益提升了 8 倍，同时仍保持完整 BTC 敞口。

### 为什么美元计价债务很重要

这种设计可以保护 Vault 免于被清算：

- **借入资金保持为 USDC**：Vault 不会把借来的稳定币再兑换成 BTC 或 ETH 这类波动资产
- **债务始终有覆盖**：即便你的抵押品（cbBTC、ETH）下跌 50%，借来的美元部分仍保持价值
- **不会形成死亡螺旋**：借来的那部分不会随着价格下跌放大亏损

**示例**：仍然以刚才那笔 100,000 美元的 cbBTC 存款为例：
- Vault 借出 70,000 美元 GREEN（70% LTV），再换成 USDC
- cbBTC 下跌 40% → 你的原始抵押品现在只值 60,000 美元
- 但那 70,000 美元借款部分依然以 USDC 形式在持续生息
- 这些 USDC 同样被作为抵押品存回 Ripe，因此你总共有 60,000 + 70,000 = 130,000 美元抵押品，对应 70,000 美元债务
- 头寸仍然健康，因为借来的部分没有贬值，而且还额外补充了抵押品

这与传统杠杆头寸不同。传统做法会把借来的钱继续买入波动资产，而那样的结构更容易在价格下跌时连环清算。

### AI 在做什么

AI 同时管理收益与风险：

- **抵押品管理**：先把资产存入 Earn Vault，再存入 Ripe Protocol
- **策略性借款**：以抵押品借出 GREEN，然后要么兑换成 USDC（如果 DEX 流动性更优），要么直接作为 Savings GREEN 进行质押，总之借来的部分仍会继续生息
- **只保留美元敞口**：借来的资金始终保持在美元计价资产中（USDC 或 GREEN），不会被换成波动资产
- **债务比率监控**：维持可配置的最大杠杆水平
- **自动去杠杆**：当债务比率接近上限时自动降低杠杆

### 风险管理

杠杆 Vault 内置了多层安全机制：

| 控制项 | 作用 |
|---------|---------|
| **仅限美元借款** | 借来的资金保持稳定币敞口 |
| **最大债务比率** | 对杠杆设置硬上限（例如最多 200%） |
| **自动去杠杆** | 比率接近上限时主动降低头寸 |
| **提现安全** | 为赎回自动拆解杠杆 |
| **持续监控** | AI 全天候跟踪健康因子 |

### 示例：杠杆 cbBTC Vault

**你存入 1 枚 cbBTC（价值 100,000 美元）**：

```
你的 1 枚 cbBTC
    ↓
cbBTC Earn Vault（赚取 0.5%）
    ↓
把 Vault 份额作为抵押品存入 Ripe
    ↓
借出 70,000 美元 GREEN（70% LTV）→ 兑换成 USDC
    ↓
USDC Earn Vault（赚取 8%）
    ↓
把 USDC Vault 份额再存回 Ripe 作为额外抵押品
```

**收益放大效果**：
- 你的 cbBTC 在 cbBTC Earn Vault 中赚取 0.5% → 每年 +500 美元
- 借款成本按 3%（折扣利率）计算 → 每年 -2,100 美元
- 借来的 70,000 美元在 USDC Earn Vault 中赚取 8% → 每年 +5,600 美元
- **净收益：4.0%**，相较基础收益约提升 8 倍

**如果 cbBTC 下跌 40%**：
- 你的 cbBTC 抵押品：现值 60,000 美元
- 你的 USDC 抵押品：仍为 70,000 美元
- 总抵押品：130,000 美元，对应 70,000 美元债务
- 头寸仍然健康，没有清算风险

**你的风险敞口**：仅来自原始存入资产（cbBTC）的价格波动。杠杆不会放大你的下行亏损，它只会放大收益。如果 cbBTC 下跌 40%，你的初始价值下跌 40%，与直接持有 cbBTC 相同。借来的美元部分保持原值，不会增加额外的下行风险。

## 存款与份额

### 如何存入

1. 授权 Vault 合约使用你的代币
2. 调用 `deposit()` 并传入目标金额
3. 在钱包中收到 Vault 份额

**份额价格计算公式**：
```
份额价格 = Vault 总资产 / 已发行总份额
```

首次存入时，1 份额 = 1 单位资产。随着 Vault 持续赚取收益，份额价格会逐步上升。

### 如何理解你的份额

Vault 份额代表你对 Vault 全部资产的按比例所有权：

- **100 份额，价格 1.00 美元** = 你对 Vault 资产拥有 100 美元的索取权
- **Vault 获得 10% 收益** = 份额价格上升到 1.10 美元
- **你的 100 份额现在价值 110 美元** = 你已经捕捉到了收益

你不需要手动领取奖励，也不需要自己做再平衡。你的份额会自动反映 Vault 的整体表现。

### 提取

你可以随时把份额赎回为底层资产：

1. 调用 `redeem()` 或 `withdraw()`，传入你的份额数量
2. AI 会在需要时自动拆解底层头寸
3. 底层资产返回到你的钱包

系统会保留一个小缓冲（2%），以确保即使在拆解头寸时，你也能收到预期金额。

## Vault 安全性与控制

### 只允许已批准协议

每个 Vault 都运行在严格边界内，而且这些边界由智能合约强制执行，而不是由公司政策约束：

- **白名单目标地址**：AI 只能把资金存入预先批准的收益协议
- **由注册表管理**：Underscore 治理负责维护已批准协议列表
- **不会出现野生存款**：AI 从代码层面就无法把资金转到未批准的目标地址，这不是“我们承诺”，而是“代码做不到”

这些规则都写在链上。不会有野路子交易，不会去实验性农场，也不会出现“AI 幻觉”。

### 存款控制

Vaults 还可以加入额外保护措施：

| 控制项 | 作用 |
|---------|---------|
| **存款上限** | 限制总资产规模，避免过度集中 |
| **Allowlist 模式** | 仅允许已批准地址存款 |
| **暂停存款** | 在需要时暂时停止新存款 |

### 紧急控制

面对极端市场情况时：

- **冻结操作**：治理可以暂停 Vault 的全部活动
- **暂停提现**：在市场压力过大时临时冻结提现（仅治理可用）
- **资产仍受保护**：即便被冻结，你对份额的索取权依然有效

## 业绩费

Vault 只会对利润收费，绝不会对你的本金收费：

- **费率**：所产生收益的 20%
- **适用对象**：仅利润，不包含存入本金
- **透明追踪**：费用会与份额机制分开记录
- **没有隐藏扣费**：你的提现结果会真实反映资产价值

**示例**：
- Vault 赚取了 1,000 美元收益
- 20% 业绩费 = 200 美元归协议
- 800 美元通过份额价格上涨分配给存款人

关于这些费用最终如何被使用，可查看[协议经济模型](protocol-economics.md)。

## Earn Vaults 与可编程钱包的区别

| 功能 | **Earn Vaults** | **可编程钱包** |
|---------|------------|------------------|
| **所有权形式** | 按比例持有份额（代币） | 对资产拥有完整直接控制权 |
| **管理方式** | 由 AI 管理，近乎免操作 | 由你和你的管理员共同决定 |
| **可定制性** | 每个 Vault 的策略固定 | 可完全自定义 |
| **可组合性** | 高（标准 ERC-20） | 较低（资产留在钱包中） |
| **准入门槛** | 任何人都可存入 | 需要先部署钱包 |
| **是否适合做抵押品** | 是（份额本身就是代币） | 更复杂 |
| **最适合** | 被动收益、可组合性 | 主动 DeFi、自定义策略 |

**适合使用 Earn Vaults 的情况**：你想获得“存进去就不用管”的收益、需要可组合/可生息抵押品，或希望不部署钱包也能参与。

**适合使用可编程钱包的情况**：你想拥有完整控制、自定义策略、代理执行、支付自动化，或需要精确限额下的管理员委托。

## 常见问题

**我可以随时提现吗？**
可以。提现是即时的。Vault 会在单笔交易中自动拆解所需头寸，以满足你的请求。

**如果 Vault 亏钱了怎么办？**
你的份额会反映 Vault 的真实价值。如果底层资产因为市场变化而贬值（而不是 AI 出错），份额价格就会下跌。业绩费只会对利润收取。

**我可以存入哪些资产？**
每个 Vault 都会指定自己的底层资产（例如 USDC、ETH 等）。你存入该资产，并获得该 Vault 对应的份额代币。

**Vault 份额可以转让吗？**
可以。Vault 份额是标准 ERC-20 代币。你可以转移、交易，或在任何接受 ERC-20 的协议里把它当作抵押品使用。

**最低存款额是多少？**
协议层面没有强制的最低限额，但由于 Gas 成本存在，过小的存款通常不划算。
